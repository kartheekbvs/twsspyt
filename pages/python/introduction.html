Introduction to Python

Python is a high-level, interpreted, general-purpose programming language. This chapter covers what Python is, why it's powerful, its key features, and how to write your very first programs — directly from leading textbooks.

1 · What is Python?

Python is a high-level, interpreted, dynamically typed, general-purpose programming language. It was designed with an emphasis on code readability and the principle that there should be one — and preferably only one — obvious way to do something.

The key distinction between Python and languages like C or Java is that Python code reads almost like plain English. Python handles memory management automatically, and its syntax is minimal and expressive.

💡
Textbook Definition

"Python is an interpreted language, which means it is not compiled directly to machine code but is instead run by another program (the Python interpreter). This makes development faster but typically slower at runtime than compiled languages." — Python for Data Analysis, Wes McKinney

2 · History & Evolution

Python was created by Guido van Rossum, a Dutch programmer, as a successor to the ABC language. The name was inspired not by the snake, but by the British comedy series Monty Python's Flying Circus.

Version Year Key Highlights
Python 1.0 1994 First release: lambda, map, filter, reduce
Python 2.0 2000 List comprehensions, garbage collection
Python 3.0 2008 Major redesign: print(), unicode, int division
Python 3.10 2021 Structural pattern matching (match/case)
Python 3.12 2023 Performance improvements, f-string improvements
⚠️
Python 2 is Dead

Python 2 reached end-of-life on January 1, 2020. All modern code should use Python 3.x.

3 · Key Features of Python

  • Interpreted: Code is executed line-by-line at runtime. No compilation step.
  • Dynamically Typed: Variable types are determined at runtime. x = 5 is valid — no int x = 5 needed.
  • Strongly Typed: Unlike JavaScript, Python won't silently convert types. "5" + 5 raises a TypeError.
  • Object-Oriented: Everything in Python is an object — numbers, strings, functions.
  • Batteries Included: Huge standard library (os, json, datetime, http, etc.)
  • Cross-Platform: Runs on Windows, macOS, Linux without modification.
  • Garbage Collection: Memory managed automatically using reference counting + cyclic GC.
  • Multiple Paradigms: Supports procedural, object-oriented, and functional programming.

4 · How Python Works (Execution Model)

When you run a Python script, several things happen under the hood:

  1. Lexical Analysis (Tokenizing): The Python interpreter reads your source code and breaks it into tokens (keywords, identifiers, operators).
  2. Parsing: Tokens are organized into an Abstract Syntax Tree (AST) — a tree representation of your code's structure.
  3. Compilation to Bytecode: The AST is compiled to Python bytecode (.pyc files stored in __pycache__/).
  4. Execution by CPython VM: The Python Virtual Machine (PVM) — typically CPython — reads and executes the bytecode instruction by instruction.
Python — Under the Hood 
# You can inspect Python bytecode with the `dis` module
import dis

def add(a, b):
    return a + b

dis.dis(add)

# OUTPUT (bytecode instructions):
#   2           0 LOAD_FAST                0 (a)
#               2 LOAD_FAST                1 (b)
#               4 BINARY_ADD
#               6 RETURN_VALUE
▶ Output
2 0 LOAD_FAST 0 (a)
2 LOAD_FAST 1 (b)
4 BINARY_OP 0 (+)
6 RETURN_VALUE
🔁 How It Works

dis.dis() disassembles a function to show the bytecode Python actually runs. Each line is an instruction for the Python Virtual Machine. LOAD_FAST loads a local variable, BINARY_ADD pops two values and pushes the sum.

5 · Your First Python Programs

Let's write the classic "Hello, World!" and explore what each part does:

Python · hello_world.py 
# This is a comment — Python ignores it
print("Hello, World!")                  # Calls the built-in print() function
print("Python version:", 3.12)          # Multiple arguments
print("Sum of 5 + 3 =", 5 + 3)         # Expression inside print()
▶ Output
Hello, World!
Python version: 3.12
Sum of 5 + 3 = 8
🔁 Return Value of print()

print() always returns None. It is called purely for its side effect — writing text to the standard output stream (stdout). The return value is almost never used.

result = print("Hello") → result is None

Basic Variable Assignment

Python · variables_intro.py 
# Variables — no type declarations needed
name = "Alice"            # str
age = 25                  # int
height = 1.75             # float
is_student = True         # bool

# f-strings for formatted output (Python 3.6+)
print(f"Name: {name}, Age: {age}, Height: {height}m")

# Check the type of any variable
print(type(name))          # <class 'str'>
print(type(age))           # <class 'int'>
print(type(height))        # <class 'float'>
print(type(is_student))    # <class 'bool'>
▶ Output
Name: Alice, Age: 25, Height: 1.75m
<class 'str'>
<class 'int'>
<class 'float'>
<class 'bool'>

8 · Installation & First Run

Installing Python

  1. Download Python from python.org/downloads
  2. Run the installer — check "Add Python to PATH"
  3. Verify installation in terminal:
Terminal / Shell 
python --version
# Python 3.12.0

python -c "print('Python works!')"
# Python works!

# Launch interactive REPL
python
# >>> 2 + 2
# 4
💡
Use Virtual Environments

Always create a virtual environment for each project to avoid dependency conflicts: python -m venv myenv → myenv\Scripts\activate (Windows)
