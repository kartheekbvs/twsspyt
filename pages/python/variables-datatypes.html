Variables & Data Types

In Python, a variable is a named reference to a value stored in memory. Python has a rich set of built-in data types covering numbers, text, booleans, and special values. This chapter covers every built-in type with full examples, type conversions, and how Python manages memory.

1 · Variable Assignment & Naming

Variables in Python are created when you assign a value to a name. There is no var, let, or int keyword needed.

Python · Assignment 
# Simple assignment
x = 10
name = "Alice"
price = 99.99

# Multiple assignment on one line
a, b, c = 1, 2, 3
print(a, b, c)    # 1 2 3

# Swap variables (Python idiom — no temp variable needed)
a, b = b, a
print(a, b)       # 2 1

# Augmented assignment
count = 0
count += 1        # count = count + 1 → 1
count *= 5        # count = count * 5 → 5

# Chain assignment (all point to the same object)
x = y = z = 0
print(x, y, z)    # 0 0 0
▶ Output
1 2 3
2 1
0 0 0

Naming Rules & Conventions

2 · Integers (int)

Python integers are arbitrary precision — they can be as large as memory allows. There is no integer overflow in Python.

Python · Integers 
# Standard integers
x = 42
big = 10 ** 100        # A googol! Python handles huge numbers
negative = -17

# Different numeric bases
binary = 0b1010       # Binary  → 10
octal  = 0o17         # Octal   → 15
hex_   = 0xFF         # Hex     → 255

print(binary, octal, hex_)

# Underscores for readability (Python 3.6+)
million = 1_000_000
print(million)        # 1000000

# Integer operations
print(17 // 3)        # Floor division → 5
print(17 % 3)         # Modulo        → 2
print(2 ** 8)         # Power         → 256

# Built-in int functions
print(abs(-42))       # 42
print(bin(255))       # '0b11111111'
print(hex(255))       # '0xff'
▶ Output
10 15 255
1000000
5
2
256
42
0b11111111
0xff
🔁 Return Values

abs(-42) → 42 (int)  |  bin(255) → '0b11111111' (str)  |  hex(255) → '0xff' (str)

3 · Floating-Point Numbers (float)

Python floats follow the IEEE 754 double-precision standard (64-bit). This gives approximately 15-17 significant digits of precision.

Python · Floats 
x = 3.14
y = 2.5e3          # Scientific notation → 2500.0
z = 1.5e-4         # → 0.00015

# ⚠ Floating-point precision issue (IEEE 754)
print(0.1 + 0.2)    # 0.30000000000000004  (NOT 0.3!)

# Fix: use round() or the decimal module
print(round(0.1 + 0.2, 2))   # 0.3

from decimal import Decimal
print(Decimal('0.1') + Decimal('0.2'))  # 0.3 (exact)

# Special float values
import math
print(math.inf)         # inf
print(-math.inf)        # -inf
print(math.nan)         # nan
print(math.isnan(math.nan))  # True

# float functions
print(round(3.14159, 2))  # 3.14
print(int(3.9))           # 3  (truncates, not rounds)
▶ Output
0.30000000000000004
0.3
0.3
inf
-inf
nan
True
3.14
3
⚠️
IEEE 754 Floating-Point

Never compare floats with ==. Use abs(a - b) < 1e-9 or math.isclose(a, b) instead.

4 · Booleans (bool)

Python's bool is a subclass of int. True is literally 1 and False is 0.

Python · Booleans 
# bool is a subclass of int!
print(True + True)   # 2
print(True * 5)     # 5
print(int(True))    # 1
print(int(False))   # 0

# Truthy and Falsy values:
# Falsy: False, None, 0, 0.0, "", [], {}, set()
# Truthy: everything else
print(bool(0))       # False
print(bool(""))      # False
print(bool([]))      # False
print(bool(42))      # True
print(bool("hi"))   # True

# Boolean operators
print(True and False)   # False
print(True or False)    # True
print(not True)          # False

5 · NoneType (None)

None is Python's null value. It represents the absence of a value. It's the only instance of NoneType.

Python · None 
result = None
print(result)              # None
print(type(result))       # <class 'NoneType'>

# Always use 'is' to check for None, not '=='
if result is None:
    print("No value assigned")

# Functions without return statement return None
def do_something():
    x = 5 + 3              # No return

val = do_something()
print(val)                 # None
print(val is None)        # True
▶ Output
None
<class 'NoneType'>
No value assigned
None
True

6 · Type Conversion (Casting)

Python · Type Casting 
# int() — convert to integer
print(int("42"))       # 42 (str → int)
print(int(3.9))        # 3  (float → int, truncates!)
print(int(True))       # 1  (bool → int)

# float() — convert to float
print(float("3.14"))   # 3.14
print(float(5))        # 5.0

# str() — convert anything to string
print(str(42))         # '42'
print(str(True))       # 'True'
print(str([1,2,3]))   # '[1, 2, 3]'

# bool() — convert to boolean
print(bool(1))         # True
print(bool(0))         # False

# Errors: invalid conversions
try:
    int("hello")       # ValueError!
except ValueError as e:
    print(f"Error: {e}")

7 · How Python Stores Variables in Memory

In Python, variables are not boxes containing values — they are labels (references) pointing to objects in memory.

Python · Memory & References 
# id() returns the memory address of an object
x = 42
y = 42
print(id(x) == id(y))   # True  (Python caches small ints!)

# 'is' checks identity (same object), '==' checks equality (same value)
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)           # True  (same values)
print(a is b)           # False (different objects in memory)

# Aliasing — both point to same object
c = a
c.append(4)
print(a)                # [1, 2, 3, 4] — 'a' was modified through 'c'!

# To avoid aliasing, make a copy
d = a.copy()            # or a[:]
d.append(5)
print(a)                # [1, 2, 3, 4] — 'a' unchanged
▶ Output
True
True
False
[1, 2, 3, 4]
[1, 2, 3, 4]
🔑
Key Insight: Integer Caching

Python caches integers from -5 to 256 for performance. So x = 42; y = 42 — both x and y point to the same object. For integers outside this range, Python creates new objects, so is may return False.
