Operators

Operators are symbols that perform operations on values and variables. Python provides arithmetic, comparison, logical, assignment, bitwise, identity, and membership operators — each with defined precedence rules.

1 · Arithmetic Operators

Operator Name Example Result
+ Addition 5 + 3 8
- Subtraction 5 - 3 2
* Multiplication 5 * 3 15
/ True Division 7 / 2 3.5
// Floor Division 7 // 2 3
% Modulus 7 % 2 1
** Exponentiation 2 ** 8 256
Python · arithmetic.py 
a, b = 17, 5
print(a + b)    # 22
print(a - b)    # 12
print(a * b)    # 85
print(a / b)    # 3.4  — always float in Python 3
print(a // b)   # 3    — floor division
print(a % b)    # 2    — remainder
print(a ** b)   # 1419857

# Floor division with negatives rounds toward -infinity
print(-7 // 2)  # -4  (not -3)
print(7 // -2)  # -4  (not -3)
▶ Output
22 · 12 · 85 · 3.4 · 3 · 2 · 1419857 · -4 · -4
🔁 Key: / vs //

/ always returns float (e.g., 4/2 == 2.0). Use // for integer floor division. Breaking change from Python 2.

2 · Comparison Operators

Always return bool. Python uniquely supports chained comparisons.

Operator Meaning Example Result
== Equal 5 == 5 True
!= Not equal 5 != 3 True
> Greater than 5 > 3 True
< Less than 3 < 5 True
>= Greater or equal 5 >= 5 True
<= Less or equal 3 <= 4 True
Python · comparison.py 
x = 10
print(x == 10)          # True
print(x != 5)           # True

# Chained comparisons — unique to Python!
age = 25
print(18 <= age < 65)   # True  (checks both conditions)
print(1 < 2 < 3 < 4)   # True

# String comparison is lexicographic (by ASCII value)
print("apple" < "banana")  # True ('a' < 'b')
▶ Output
True · True · True · True · True

3 · Logical Operators

Logical operators use short-circuit evaluation and return the actual value (not just True/False).

Operator Description Short-circuits at
and True if both operands are truthy First falsy value
or True if any operand is truthy First truthy value
not Reverses truthiness Always evaluates
Python · logical.py 
print(True and False)   # False
print(True or False)    # True
print(not True)         # False

# or returns first truthy value (not just bool)
print(0 or "hello")     # 'hello'
print("hi" or "bye")    # 'hi'
print(None or 42)       # 42

# Common pattern: default value
name = None
display = name or "Anonymous"
print(display)          # Anonymous

# and returns first falsy, or last if all truthy
print(1 and 2 and 3)   # 3  (all truthy, returns last)
print(1 and 0 and 3)   # 0  (0 is falsy)
▶ Output
False · True · False · hello · hi · 42 · Anonymous · 3 · 0

# Walrus operator (:=) — assign and use in same expression
data = [1, 2, 3, 4, 5]
if (n := len(data)) > 3:
    print(f"List has {n} items — processing...")
# Output: List has 5 items — processing...

# Useful in while loops
import io
stream = io.StringIO("hello world")

5 · Bitwise Operators

Work on integers at the binary bit level. Common in flags, masks, and performance-critical code.

Operator Name a=0b1010, b=0b1100 Result
& AND a & b 0b1000 = 8
| OR a | b 0b1110 = 14
^ XOR a ^ b 0b0110 = 6
~ NOT ~a -11
<< Left shift a << 1 20
>> Right shift a >> 1 5
Python · bitwise.py 
a = 0b1010   # 10
b = 0b1100   # 12

print(bin(a & b))   # 0b1000 (8)
print(bin(a | b))   # 0b1110 (14)
print(bin(a ^ b))   # 0b110  (6)
print(~a)           # -11  (two's complement: -(n+1))

# Shifts = fast multiply/divide by 2
print(8 << 1)   # 16  (8 * 2)
print(8 >> 1)   # 4   (8 / 2)
print(1 << 3)   # 8   (1 * 2^3 = 8)
▶ Output
0b1000 · 0b1110 · 0b110 · -11 · 16 · 4 · 8

6 · Identity & Membership Operators

is / is not — Identity

Tests if two variables point to the same object in memory, not equal value.

Python · identity_membership.py 
a = [1, 2, 3]
b = a           # same object
c = [1, 2, 3]   # same value, different object

print(a == c)   # True  — equal value
print(a is c)   # False — different objects
print(a is b)   # True  — same object

# Always use 'is' for None checks
x = None
print(x is None)   # True ✓ (preferred over == None)

# Membership: 'in' and 'not in'
fruits = ["apple", "banana", "cherry"]
print("apple" in fruits)        # True
print("grape" not in fruits)    # True

# Works on strings, dicts (checks keys), sets
print("py" in "python")                  # True
print("name" in {"name": "Alice"})        # True
▶ Output
True · False · True · True · True · True · True · True
⚠️
== vs is

Never use is to compare integers or strings for equality — only for None, True, False. CPython caches small ints (-5 to 256) which can give misleading results with is.

7 · Operator Precedence (High → Low)

Priority Operators Note
1 (highest) () Parentheses
2 ** Exponentiation (right-assoc)
3 +x, -x, ~x Unary operators
4 *, /, //, % Multiplication/division
5 +, - Addition/subtraction
6 <<, >> Bit shifts
7 & Bitwise AND
8 ^ Bitwise XOR
9 | Bitwise OR
10 ==, !=, <, >, is, in ... Comparisons
11 not Logical NOT
12 and Logical AND
13 (lowest) or Logical OR
Python · precedence.py 
print(2 + 3 * 4)       # 14  (* before +)
print((2 + 3) * 4)     # 20  (parens override)
print(2 ** 3 ** 2)     # 512  (right-assoc: 2**(3**2)=2**9)
print(not False and True or False)  # True
# Evaluates: (not False) and True or False
#          = True and True or False
#          = True or False → True
▶ Output
14 · 20 · 512 · True
💡
Best Practice

Use parentheses to make precedence explicit in complex expressions. Don't rely on memorising the full precedence table — readability matters more.
